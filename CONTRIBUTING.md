# Contributing to ShapeShred

شكراً لاهتمامك بالمشاركة في تطوير ShapeShred! نحن نرحب بجميع المساهمات.

## كيفية المساهمة

### الإبلاغ عن المشاكل

إذا وجدت مشكلة، يرجى فتح issue على GitHub مع:
- وصف واضح للمشكلة
- خطوات لإعادة إنتاج المشكلة
- السلوك المتوقع
- لقطات شاشة (إن أمكن)
- معلومات عن البيئة (OS, Node.js version, إلخ)

### طلبات الميزات

لطلب ميزة جديدة، يرجى:
- فتح issue على GitHub
- وصف الميزة المطلوبة بوضوح
- شرح سبب أهمية هذه الميزة
- اقتراح حلول محتملة

### إرسال Pull Requests

1. Fork المشروع
2. أنشئ branch جديد (`git checkout -b feature/amazing-feature`)
3. Commit التغييرات (`git commit -m 'Add amazing feature'`)
4. Push إلى branch (`git push origin feature/amazing-feature`)
5. افتح Pull Request

## معايير الكود

### TypeScript
- استخدم TypeScript لجميع الملفات الجديدة
- اتبع معايير ESLint المحددة
- أضف types لجميع الدوال والمتغيرات

### التنسيق
- استخدم Prettier لتنسيق الكود
- اتبع نمط الكود الموجود
- استخدم أسماء متغيرات واضحة ووصفية

### التوثيق
- أضف JSDoc comments للدوال المعقدة
- حدّث README إذا أضفت ميزات جديدة
- أضف أمثلة الاستخدام

## اختبارات

- تأكد من اجتياز جميع الاختبارات قبل إرسال PR
- أضف اختبارات للميزات الجديدة
- حافظ على تغطية الاختبارات عالية

## رسائل Commit

استخدم رسائل commit واضحة وموصوفة:
```
feat: add user authentication
fix: resolve database connection issue
docs: update API documentation
refactor: improve code structure
test: add unit tests for auth service
```

## مراجعة الكود

جميع Pull Requests تخضع للمراجعة. يرجى:
- أن تكون صبوراً خلال عملية المراجعة
- الرد على التعليقات والأسئلة
- إجراء التعديلات المطلوبة

## السلوك

- كن محترماً ومتعاوناً
- ركز على ما هو الأفضل للمجتمع
- احترم آراء الآخرين

## الأسئلة؟

إذا كانت لديك أي أسئلة، يرجى فتح issue أو التواصل معنا عبر:
- البريد الإلكتروني: contact@shapeshred.com
- Discord: [رابط Discord]
